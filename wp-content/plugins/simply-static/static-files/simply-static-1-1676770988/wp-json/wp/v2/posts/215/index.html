    Applications that are accessed using Web-based protocols (HTTP\/S) are known as Web Applications.<\/p>\n\n\n\n

      Front End (UI)<\/h3><\/div>\n\n\n\n

      Front End is the User Interface (UI) and, it’s the visual part of a Web Application.<\/p>\n\n\n\n

      Front End is mostly accessed from Web Browsers.<\/p>\n\n\n\n

      Web Browsers contain 2 engines:<\/p>\n\n\n\n

      • Browser Engine<\/li>
      • JavaScript Engine<\/li><\/ul>\n\n\n\n

        JavaScript Engine<\/h3><\/div>\n\n\n\n

        A Browser Engine renders the web page, but it can’t handle all events that may occur.<\/p>\n\n\n\n

        A JavaScript Engine is for this purpose.<\/p>\n\n\n\n

        Browser Engine<\/h3><\/div>\n\n\n\n

        Websites are divided into multiple pages. At any time a single page is rendered. This is called a Web Page.<\/p>\n\n\n\n

        Browser Engines are responsible to parse a web page and, rendering it in browser.<\/p>\n\n\n\n

        There exists these 3 popular languages for Front End Development.<\/p>\n\n\n\n

        Let’s see how they operate<\/p>\n\n\n\n

        • HTML \u279f Structure<\/li>

        • JavaScript \u279f Behaviour <\/li><\/ul>\n\n\n\n
